## 1. 프로젝트 개요

### 앱 이름
**MemoryLink** (메모리링크)

### 목적
노인을 위한 종합 디지털 건강 관리 앱으로, 세 가지 핵심 기능을 제공합니다:
- **인지 훈련**: 뇌 훈련 게임으로 기억력, 계산력, 집중력 향상
- **보행 분석**: 스마트폰 센서로 걷는 패턴을 분석해 치매 위험도 평가
- **건강 리포트**: 훈련 결과와 보행 데이터를 종합한 PDF 리포트 생성

### 기본 정보

| 항목 | 내용 |
|------|------|
| 버전 | 1.0.0 |
| 개발 언어 | Dart (Flutter 프레임워크) |
| 지원 플랫폼 | Android, iOS, Web |
| 필요 Flutter 버전 | 3.11.3 이상 |
| 데이터 저장 방식 | 로컬(SQLite) + 클라우드(Supabase) 병행 |
| 기본 테스트 계정 | 아이디: `admin` / 비밀번호: `admin` |

## 2. 개발 환경 설정 가이드

> Flutter를 처음 설치하시는 분을 위한 단계별 가이드입니다.

### 2-1. Flutter SDK 설치

1. [Flutter 공식 홈페이지](https://flutter.dev/docs/get-started/install)에서 본인의 운영체제에 맞는 SDK를 다운로드합니다.
2. 압축을 풀고 원하는 폴더에 저장합니다. (예: `C:\flutter`)
3. 시스템 환경 변수 `PATH`에 `C:\flutter\bin`을 추가합니다.
4. 터미널(명령 프롬프트)을 열고 아래 명령어로 설치를 확인합니다:
   ```
   flutter doctor
   ```
   모든 항목에 초록색 체크(✓)가 표시되면 완료입니다.

### 2-2. 프로젝트 의존성 설치

터미널에서 프로젝트 폴더로 이동 후 아래 명령어를 실행합니다:
```
cd d:\Test_Android\flutter_application_1
flutter pub get
```
이 명령어는 `pubspec.yaml`에 적힌 모든 패키지를 자동으로 다운로드합니다.

### 2-3. 앱 실행

**안드로이드 에뮬레이터로 실행:**
```
flutter run
```

**특정 기기 선택 후 실행:**
```
flutter devices          ← 연결된 기기 목록 확인
flutter run -d [기기ID]  ← 원하는 기기로 실행
```

### 2-4. 처음 실행 시 로그인

앱을 처음 실행하면 로그인 화면이 나타납니다.
- **아이디:** `admin`
- **비밀번호:** `admin`

이 계정은 앱 최초 실행 시 자동으로 생성되는 기본 테스트 계정입니다.

## 5. 핵심 파일 상세 설명 (core/ 폴더)

### 5-1. `main.dart` — 앱 시작점

**무슨 역할인가요?**
앱을 켜면 가장 먼저 실행되는 파일입니다. 앱에 필요한 모든 것을 준비하고, 화면을 띄웁니다.

**하는 일:**
1. 한국어 날짜 형식 초기화
2. 백그라운드 만보계 서비스 시작
3. 클라우드(Supabase) 연결
4. 앱 전체에서 사용할 데이터 관리자(Provider)들을 등록
5. 앱 화면 표시 시작

**등록되는 데이터 관리자(Provider) 목록:**

| Provider | 역할 |
|----------|------|
| `UserProvider` | 로그인한 사용자 정보 관리 |
| `GaitProvider` | 보행 분석 데이터 관리 |
| `PedometerManager` | 걸음 수 및 활동량 관리 |
| `SettingsProvider` | 글자 크기, 음성 안내 등 앱 설정 |
| `DifficultyProvider` | 게임 난이도 자동 조정 |

---

### 5-2. `core/router.dart` — 화면 경로 설정

**무슨 역할인가요?**
앱 내 모든 화면의 "주소"를 정의합니다. 웹사이트의 URL과 비슷한 개념입니다.

**정의된 화면 경로:**

| 경로 | 화면 | 설명 |
|------|------|------|
| `/login` | 로그인 화면 | 앱 시작 시 기본 화면 |
| `/register` | 회원가입 화면 | 새 계정 만들기 |
| `/onboarding` | 온보딩 화면 | 처음 사용 시 목표 설정 |
| `/` | 메인 화면 | 5탭 하단 메뉴 |
| `/training_hub` | 훈련 허브 | 7가지 게임 목록 |
| `/game/comparison` | 비교 게임 | 숫자 비교 게임 |
| `/game/sequence` | 수열 게임 | 패턴 찾기 게임 |
| `/gait` | 보행 분석 | 걸음 분석 화면 |
| `/assessment` | 건강 평가 | 설문 및 인지 과제 |
| `/profile` | 프로필 | 개인정보 및 설정 |
| `/reports` | 리포트 | 주간 건강 리포트 |

**자동 리디렉션:** 로그인하지 않은 상태에서 다른 화면에 접근하면 자동으로 `/login`으로 이동합니다.

---

### 5-3. `core/user_provider.dart` — 사용자 상태 관리

**무슨 역할인가요?**
로그인한 사용자의 모든 정보를 앱 전체에서 사용할 수 있도록 관리합니다.

**관리하는 정보:**
- 로그인/로그아웃 상태
- 개인 정보 (나이, 체중, 혈액형, 복용 약물)
- 인지 훈련 점수 (계산력, 논리력, 기억력, 집중력 각 0~10점)
- 설문 응답 내용
- 온보딩 완료 여부

**주요 기능(함수):**

| 함수명 | 하는 일 |
|--------|--------|
| `login(아이디, 비밀번호)` | 로그인 처리 |
| `register(아이디, 비밀번호, 나이, 체중, 목표)` | 회원가입 처리 |
| `checkLoginStatus()` | 앱 시작 시 자동 로그인 확인 |
| `updateMedicalInfo()` | 건강 정보 업데이트 |
| `setCognitiveScore(카테고리, 점수)` | 훈련 결과 저장 |

**데이터 보존 방법:**
- 로그인 정보 → `SharedPreferences`에 저장 (앱 껐다 켜도 유지)
- 훈련 점수 → `SQLite DB`에 저장

---

### 5-4. `core/database_helper.dart` — 로컬 데이터베이스

**무슨 역할인가요?**
스마트폰 내부에 데이터를 저장하고 불러오는 창고 역할을 합니다. 인터넷 없이도 작동합니다.

**데이터베이스 파일명:** `memorylink.db`

**저장되는 테이블(표) 목록:**

| 테이블 | 저장 내용 |
|--------|----------|
| `users` | 사용자 계정 (아이디, 비밀번호, 나이, 체중, 목표 등) |
| `training_scores` | 인지 훈련 점수 (계산력, 논리력, 기억력, 집중력) |
| `daily_steps` | 일별 활동량 (걸음 수, 칼로리, 이동 거리) |
| `checklist` | 일일 할 일 체크 |

**자동 생성 계정:** 앱 최초 실행 시 `admin/admin` 계정이 자동으로 만들어집니다.

### 5-5. `core/theme.dart` — 디자인 시스템

**무슨 역할인가요?**
앱 전체의 색상, 폰트, 버튼 모양 등 시각적 요소를 한 곳에서 정의합니다.
여기서 색상 하나를 바꾸면 앱 전체에 반영됩니다.

**색상 테마:**

| 역할 | 색상 | 색상코드 |
|------|------|---------|
| 주요 색상 (Primary) | 딥 청록색 | `#006064` |
| 보조 색상 (Secondary) | 세이지 그린 | `#81C784` |

### 6-5. 보행 분석 (gait_analysis/) — 걸음 패턴 분석

**포함된 화면:**
- `GaitScreen` — 수동 보행 측정 세션 화면
- `PreciseGaitAnalysisScreen` — 상세 보행 바이오마커 분석
- `WalkingDashboardScreen` — 일별/주별 활동량 차트

**보행 분석 원리:**

1. 스마트폰 가속도계가 움직임 데이터를 수집합니다
2. `GaitAnalyzer`가 데이터를 분석해 발걸음을 감지합니다
3. 걸음 간격의 일관성(변동 계수)을 계산합니다
4. 변동 계수가 클수록 보행이 불안정 → 치매 위험 신호 가능성

**계산되는 보행 지표:**

| 지표 | 의미 |
|------|------|
| 보폭 간격 | 발걸음 사이의 시간 (300ms~2000ms 정상 범위) |
| 보행 변동성 | 보폭의 불규칙 정도 (낮을수록 안정적) |
| 걸음 수 | 총 걸음 수 |

**일일 활동량 계산식:**
- 이동 거리: `걸음 수 × 0.7 ÷ 1000` (km)
- 소모 칼로리: `걸음 수 × 0.04 × (체중/60) × 나이보정계수` (kcal)

## 7. 데이터 흐름 설명

### 7-1. 로그인 흐름

```
사용자가 아이디/비밀번호 입력
         ↓
   UserProvider.login()
         ↓
   DatabaseHelper.getUser()
   (SQLite에서 사용자 조회)
         ↓
  로그인 성공 → SharedPreferences에 저장 (자동 로그인용)
         ↓
     MainNavScreen (메인 화면 이동)
```

### 7-2. 훈련 게임 점수 저장 흐름

```
사용자가 게임 플레이
         ↓
    게임 종료 후 점수 계산
         ↓
  DifficultyProvider.updatePerformance()
  (최근 5회 성적으로 다음 게임 난이도 조정)
         ↓
  UserProvider.setCognitiveScore()
         ↓
  DatabaseHelper.insertScore()
  (SQLite에 점수 기록 저장)
         ↓
  ReportsScreen에서 차트로 확인 가능
```

### 7-3. 보행 분석 흐름

```
앱 시작 시 BackgroundService 시작
         ↓
  Pedometer(만보계) 스트림 → 실시간 걸음 수 수집
  Accelerometer(가속도계) → 50개 샘플씩 수집
         ↓
       PedometerManager
  (걸음 수, 칼로리, 거리 계산 및 UI에 표시)
         ↓
       GaitProvider
  (가속도계 데이터 분석)
         ↓
       GaitAnalyzer
  (보폭 감지, 보행 변동성 계산)
         ↓
  Supabase 업로드 (로그인 상태일 때, 선택적)
```

### 7-4. 오프라인 ↔ 클라우드 데이터 흐름

```
모든 데이터는 먼저 로컬 SQLite에 저장
         ↓
  인터넷 연결 확인
         ↓
  연결됨 → Supabase에 동기화 (선택 기능)
  연결 안됨 → 로컬만 사용 (앱 정상 작동)
```

## 8. 상태 관리(Provider) 설명

### Provider 패턴이란?

앱 내 여러 화면에서 같은 데이터를 공유해야 할 때 사용하는 방법입니다.

**예시 문제:** 로그인한 사용자 이름을 홈 화면과 프로필 화면 모두에서 표시해야 합니다.

**Provider 없이:** 각 화면이 데이터를 따로 저장하고 관리 → 동기화 문제 발생

**Provider 사용 시:** 하나의 `UserProvider`가 사용자 정보를 관리하고, 모든 화면이 이것을 바라봄 → 한 곳에서 바꾸면 모든 화면 자동 업데이트

### 앱에서 사용하는 Provider 목록

| Provider 이름 | 관리하는 데이터 | 사용하는 화면 |
|--------------|--------------|-------------|
| `UserProvider` | 사용자 정보, 인지 점수 | 전체 |
| `GaitProvider` | 보행 분석 측정 상태 | 보행 관련 화면 |
| `PedometerManager` | 걸음 수, 칼로리, 거리 | 홈, 생활 화면 |
| `SettingsProvider` | 글자 크기, 음성, 진동 | 전체 |
| `DifficultyProvider` | 게임 레벨, 성적 기록 | 훈련 화면 |

### Provider 사용 방법 (Dart 코드)

```dart
// 데이터 읽기 (화면이 자동으로 새로고침됨)
final user = context.watch<UserProvider>();
Text(user.username)  // 사용자 이름 표시

// 데이터 읽기 (새로고침 불필요한 경우)
final settings = context.read<SettingsProvider>();

// 데이터 변경
context.read<UserProvider>().login('admin', 'admin');
```

**선언된 권한 목록:**

|------|------|

## 10. 자주 묻는 질문 (FAQ)

**Q. 앱 실행 시 로그인 화면이 나오는데 어떻게 하나요?**
A. 기본 테스트 계정으로 로그인하세요:
- 아이디: `admin`
- 비밀번호: `admin`

---

**Q. 인터넷 없이도 앱이 작동하나요?**
A. 네. 모든 기본 기능(훈련 게임, 보행 분석, 리포트)은 인터넷 없이 로컬에서 작동합니다.
클라우드 동기화(Supabase) 기능만 인터넷이 필요합니다.

---

**Q. 앱을 완전히 닫아도 걸음 수가 측정되나요?**
A. 안드로이드에서는 백그라운드 서비스가 계속 실행되어 걸음 수를 측정합니다.
단, 기기 설정에서 배터리 최적화가 켜져 있으면 강제 종료될 수 있습니다.
설정 → 앱 → MemoryLink → 배터리 → '제한 없음' 으로 설정하세요.

---

**Q. 훈련 게임 레벨이 자동으로 바뀌는 이유는 무엇인가요?**
A. `DifficultyProvider`가 최근 5회 게임 결과를 분석해 자동으로 난이도를 조정합니다.
정답률이 높으면 레벨이 올라가고, 낮으면 내려갑니다. (1~10단계)

---

**Q. PDF 리포트가 한글이 깨져 나오는 경우 어떻게 하나요?**
A. `assets/fonts/NanumGothic-Bold.ttf` 파일이 올바른 위치에 있는지 확인하세요.
`pubspec.yaml`에 해당 폰트가 선언되어 있어야 합니다.

---

**Q. Supabase 연결 오류가 발생하는 경우는?**
A. 현재 Supabase 클라우드 기능은 개발 중입니다. 오류가 발생해도 앱의 기본 기능은
정상 작동합니다. 오류 로그는 개발자에게 전달해 주세요.
